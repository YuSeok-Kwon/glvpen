# 비밀번호 암호화 마이그레이션 가이드

## 개요
MD5에서 BCrypt로 암호화 방식이 변경되었습니다. 기존 사용자는 비밀번호를 재설정해야 합니다.

## 변경 사항
- **이전**: MD5 해싱 (보안 취약)
- **이후**: BCrypt 암호화 (보안 강화)

## 마이그레이션 방법

### 1. 데이터베이스 사용자 비밀번호 무효화

기존 MD5로 암호화된 비밀번호는 BCrypt와 호환되지 않습니다.
다음 SQL을 실행하여 기존 비밀번호를 무효화하세요:

```sql
-- 방법 1: 비밀번호 필드를 임시 값으로 변경
UPDATE user
SET password = 'RESET_REQUIRED'
WHERE password NOT LIKE '$2a$%' AND password NOT LIKE '$2b$%';

-- 방법 2: 비밀번호를 NULL로 변경 (컬럼이 NULL 허용하는 경우)
UPDATE user
SET password = NULL
WHERE password NOT LIKE '$2a$%' AND password NOT LIKE '$2b$%';
```

> **참고**: BCrypt 암호화된 비밀번호는 `$2a$` 또는 `$2b$`로 시작합니다.

### 2. 로그인 화면 수정 (선택사항)

사용자에게 비밀번호 재설정이 필요하다는 안내를 표시할 수 있습니다:

```html
<!-- login.html에 안내 메시지 추가 -->
<div class="alert alert-info">
    <strong>안내:</strong> 보안 강화를 위해 모든 사용자의 비밀번호가 초기화되었습니다.
    비밀번호 찾기를 통해 새 비밀번호를 설정해주세요.
</div>
```

### 3. 사용자 안내 방법

**옵션 A: 이메일 발송 (권장)**
- 등록된 이메일로 비밀번호 재설정 안내 발송
- 비밀번호 찾기 링크 포함

**옵션 B: 로그인 화면 공지**
- 로그인 페이지에 팝업 또는 배너로 안내
- "보안 업데이트로 인해 비밀번호를 재설정해주세요"

**옵션 C: SMS 발송**
- 가입 시 전화번호를 수집한 경우 SMS 안내

## 비밀번호 찾기 기능 확인

비밀번호 재설정 기능이 정상 작동하는지 확인하세요:

1. `/user/find-password-view` 접속
2. 로그인 ID, 이름, 이메일 입력
3. 새 비밀번호 설정
4. 새 비밀번호로 로그인 테스트

## 질문 및 문제 해결

### Q: 기존 사용자가 로그인할 수 없어요
**A:** 비밀번호 찾기 기능을 통해 새 비밀번호를 설정하도록 안내하세요.

### Q: 비밀번호 찾기에서 이메일이 일치하지 않아요
**A:** DB의 user 테이블에서 해당 사용자의 이메일이 정확히 입력되어 있는지 확인하세요.

### Q: 개발 환경에서 테스트하려면?
**A:** 새 계정을 생성하여 BCrypt로 암호화된 비밀번호로 테스트하세요.
